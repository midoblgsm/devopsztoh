Installing a test version of concourse is straightforward. It can be done on a single machine or it can be distributed. To have a production ready system, you need to do more work.

For the sake of our course we will do the quickstart recommended by the Concourse team.

$ wget https://concourse-ci.org/docker-compose.yml
$ docker-compose up -d

The prerequisite here is to have docker and docker-compose installed in your machine. Depending on your network bandwidth, this might take some time in the first execution to download the docker images.

If everything went well you can access concourse dashboard at localhost:8080. You can log in with the username/password as test/test.

The dashboard will look like the following screenshot:

You can login and download the cli binary that will allow you to talk to concourse. After downloading the binary (called fly), please make sure to add it to your path. If you don't know how to add the binary to your path, please ask google or reach out to me.

Once your system is ready you can start interacting with concourse using fly cli.

$ fly -t devops login -c http://localhost:8080 -u test -p test
logging in to team 'main'
