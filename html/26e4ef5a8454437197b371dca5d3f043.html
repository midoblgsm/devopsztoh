First thing you need to do if you haven't done that yet, is to install a git client. You can follow this link to do so. When you are done we can get started.

Create a new repository:

$ mkdir myfirstrepo
$ cd myfirstrepo
$ git init

To checkout a repository or create a copy from it we call that "clone" the repository. You can try that locally by creating another folder and:

$ mkdir mysecondlocation
$ cd mysecondlocation
$ git clone /path/to/myfirstrepo
$ ls
myfirstrepo

For remote repository you will need to do: 

$ git clone username@host:/path/to/repo

The workflow:

The local repository consists of three trees maintained by git. The first s the working directory that holds the files. Then you have the Index that acts like a staging area. Finally you have the HEAD that points to the last commit you have made (the last change you saved using git commit.)

Any changes to your files can be staged or added to the Index using git add:

$ echo "Hello Git" > README.md
$ git add README.md

To save this change to the HEAD and make them persistent with git we need to commit the changes:

$ git commit -m "This is my first awesome commit"

Now the code is committed to the HEAD. Until now the changes are local and not shared with any remote repository.

In order to publish them we need to do a git push:

$ git push origin master

This will push the changes to the master branch in your remote repository. If you don't have one, you can add one by doing:

$ git remote add origin <server>

Afterwards the push will work.

Branches:

Developers use branches in github to develop new features that are isolated from each other. Later on these branches can get merged to master. You can create a new branch with the name of your feature:

$ git branch -b mynicefeature

Then you can do your work, add it and commit. You can push your changes to the remote repository when you are ready.

$ git push origin <branch>

You can move from one branch to another using checkout:

$ git checkout <branch>

To get the latest updates you need to do:

$ git pull

If you have done changes to another branch and want to merge it to your master. You need to checkout your master branch and do a merge.

$ git checkout master
$ git merge mynicefeature

Git will try to merge the changes from that branch to the master. If there is a conflict, you will need to fix the conflicts manually by editing the files having the conflicts. After solving the conflict, you need to mark them as merged using git add.

You can see the history of a given branch by doing a simple git log. This will show you the repository history. 

Now if you want to replace a local file by its remote version (exp: if you made a mistake and the code does not work anymore, don't panic, git got you covered), you can either replace the local changes using:

$ git checkout -- <file or folder>

Or you can be more aggressive and change all your local changes

$ git fetch origin
$ git reset --hard origin/master
