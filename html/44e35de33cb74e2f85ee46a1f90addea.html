For convenience, I created a SwaggerHub repository for our use case. You can copy the API description to Swagger Editor and change it. Try to add the missing methods, don't forget to add the needed data models. If the window is not big enough, you can jump in to https://app.swaggerhub.com/apis/midoblgsm/ShoppingCart/0.0.1.

SwaggerHub allows to manage different versions of the API. Check out the version 1.0.0 that I added. How close was it to your version?

Once you are done making your changes, you can generate the server stub or a client SDK for the API. You can also generate the fancy documentation for your API.  Go ahead and give it a try to see how close was it from our initial code.
