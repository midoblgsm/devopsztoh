Before proceeding further, if you want to run the following examples you need to have access to a Kubernetes environment.

The easiest way to have your own environment is to setup minikube. Minikube is a one node kubernetes cluster that uses virtualisation to install everything in you machine with almost all the functionality that we need to try. Please follow this link to install minikube: https://kubernetes.io/docs/tasks/tools/install-minikube/

After installing minikube you can start a cluster using the following command:

$ minikube start --memory 4096  --cpus 4

I am specifying the amount of memory and CPU to be used and this is the minimum required for the rest of our sessions. You can start this session with less resources, but going forward with istio it is gonna be a blocker.

Minikube allows you to try any feature in kubernetes and it is very easy to create and delete clusters.

After that you can start interacting with your cluster using kubernetes CLI kubectl:

$ kubectl create deployment hello-minikube --image=k8s.gcr.io/echoserver:1.10

This creates a deployment based on the specified docker image. Now we are finally ready to deploy our application to Kubernetes. Yayyy !
